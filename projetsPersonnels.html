Afficher une caméra Windows 11 sur une VM Windows 95

Technos : C, DosBox, WSL

Contexte : Réalisation d'une VM DOSBox contenant Windows 95 étant relié à Windows 11 pour transmettre un flux vidéo

Ce que j’ai fait : Le But était de faire comprendre à un dinosaure comment afficher une vidéo dans un langage compréhensible. J'ai donc codé un Serveur en C qui envoie des données retravaillées à un client sur un pont Réseau via Socket TCP.

Bilan : Cela m'a fait comprendre les limites des OS, les outils et comment les systemes communiquent et évoluent

SoundScape Manager

Technos : Java 21, JavaFX (MVC),HTTP Server, Maven, Jackson (JSON)

Contexte : Développement d'une solution complète pour Maître de Jeu (JDR) transformant un PC en table de mixage audio immersive. Le but : centraliser la gestion sonore et piloter l'ambiance à distance via smartphone.

Défis techniques relevés :
• Architecture MVC : Séparation stricte entre la logique (AudioEngine), les données (Models) et l'interface (FXML).
• Multithreading & UI : Utilisation de Task et threads pour charger les images/sons sans geler l'interface (Non-blocking UI).
• Logique Audio Avancée : Algorithme de Crossfade (transition douce) et gestion simultanée de pistes (Ambiance + Musique).
• IoT & Contrôle à distance : Création d'un serveur HTTP embarqué (sans framework lourd) exposant une API REST. Permet de contrôler le son et le volume en temps réel depuis une interface Web mobile responsive générée dynamiquement.
• Distribution : Packaging professionnel via jpackage incluant une JRE allégée pour un exécutable (.exe) ne nécessitant aucune installation préalable de Java.

Bilan : Un projet complet alliant Backend (Serveur), Frontend (JavaFX + Web Mobile) et Architecture logicielle robuste (SOLID).
